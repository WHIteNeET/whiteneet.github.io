「Note&Idea」MMOGameSystem

Posted on 2018-11-02 | In Note , Idea

一个不玩MMO的人在这扯MMO的系统……

首先，我们要明确一个问题，什么是游戏系统？
Wiki上给出PRG游戏系统的定义是这样的：

A role-playing game system is a set of game mechanics used in a role-playing game (RPG) to determine the outcome of a character’s in-game actions.

从内容上看，GameSystem is a set of game mechanics，很好，游戏系统是由一套游戏机制，这很容易理解。大多的MMORPG都会有一个复杂的游戏系统，这个系统内还有职业系统，战斗系统，经济系统，任务系统等等。但究其本源，这些系统都有其内在的运行机制。但要注意的是，机制≠规则。对于玩家而言，他们可以看到游戏内部的数值，属性，规则，然后通过这些知道这个游戏怎么玩，怎么才能玩好。但是如果我们站在一个设计者角度来看这个问题，我们从开始就已经想好了这个游戏要怎么玩，从某种意义上来讲这时游戏的机制就已经建立了，但还没有具体实现。于是，自然而然就有了后面半句话to determine the outcome of a character's in-game actions。

那么，一个游戏系统由哪些要素组成？
从组成上来讲，传统的MMORPG的游戏系统可以模块化为以下部分：
职业系统，技能系统，战斗系统，经济系统，任务系统，成长系统，道具系统，社交系统，组队系统。其中作为游戏主要部分的系统有职业，技能，战斗，经济，任务。有了这些系统，就代表着一个MMORPG已经可以运转起来给人玩了。但仅有这些是远远不够的，如果没有像成长系统，道具系统，社交系统，组队系统这类的辅助系统，这样的一个MMORPG是不足够吸引人的，换句话讲就是这样的MMORPG不够好玩。
具体案例分析先咕在这= =改天去试试基三或者魔兽再回来填。

于是，怎么构建一个游戏系统？
对于MMORPG而言，游戏的核心内容很明显，那就是角色扮演（Role-playing）和多人（Multiplayer）。藉此，我们可以确定这个框架中应该包含等级/装备成长，剧情发展，信息交流，物资流通和协作竞技这些元素，然后再细分出下级层次和衍生内容。到此为止，一个MMORPG游戏系统的基础框架（核心结构）就已经成形了。之后再对这个框架进行增删修改或者差异化构筑时就会显得很方便，同时也不会过多的影响开发方向。
